# 匿名認証を有効にする詳細手順

## 方法1: Supabaseダッシュボードで有効化（推奨）

### ステップ1: Supabaseダッシュボードにアクセス
1. [Supabase](https://supabase.com/)にアクセス
2. ログイン（アカウントがない場合は作成）

### ステップ2: プロジェクトを選択
1. ダッシュボードでプロジェクトを選択
2. 左側のメニューから **Authentication** をクリック

### ステップ3: Providers設定を開く
1. **Providers** タブをクリック
2. 一覧から **Anonymous** を探す（アルファベット順で上から）

### ステップ4: 匿名認証を有効化
1. **Anonymous** の行で、右側のトグルスイッチを **ON** にする
2. または、**Anonymous** をクリックして詳細画面を開き、**Enable** ボタンをクリック

### ステップ5: 設定を保存
- 自動的に保存されます（保存ボタンがない場合は自動保存）

### 確認方法
- **Anonymous** の行で、トグルスイッチが **ON**（緑色）になっていることを確認

## 方法2: SQLで有効化（上級者向け）

SupabaseのSQL Editorで以下を実行：

```sql
-- 匿名認証を有効化
UPDATE auth.config
SET enable_anonymous_sign_ins = true;
```

ただし、この方法はSupabaseのバージョンによっては動作しない場合があります。**方法1を推奨します**。

## トラブルシューティング

### 問題: Anonymousが見つからない
- **解決策**: Supabaseのバージョンが古い可能性があります。プロジェクトを最新バージョンにアップグレードしてください。

### 問題: 有効化してもエラーが出る
- **解決策**: 
  1. ブラウザをリロード
  2. Supabaseの設定を再度確認
  3. コンソールのエラーメッセージを確認

### 問題: 設定が保存されない
- **解決策**: 
  1. ブラウザのキャッシュをクリア
  2. 別のブラウザで試す
  3. Supabaseのサポートに問い合わせる

## 次のステップ

匿名認証を有効化したら：
1. ブラウザをリロード
2. コミュニティに参加を試みる
3. ニックネームを入力して参加

正常に動作すれば、コンソールに「✅ 匿名認証成功」と表示されます。
