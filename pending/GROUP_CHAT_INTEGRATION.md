# グループチャット統合について

## 技術的な制限

**残念ながら、LINEやFacebookなどのグループチャットで個人が送信できるメッセージに、インタラクティブなスライダーを直接埋め込むことはできません。**

### 理由

1. **LINEグループチャット**
   - 個人が送信できるメッセージタイプは限られている（テキスト、画像、動画、スタンプなど）
   - インタラクティブなUIコンポーネント（スライダーなど）は送信できない
   - Botアカウントでも、Flex Messageは送信できるが、スライダーコンポーネント自体は存在しない

2. **Facebook Messenger**
   - 個人が送信できるメッセージタイプは限られている
   - インタラクティブなUIコンポーネントはBotアカウントが必要

3. **Discord**
   - インタラクティブなコンポーネント（ボタン、スライダーなど）はBotアカウントが必要
   - 個人アカウントでは送信できない

## 現実的な代替案

### 1. URLプレビュー（OGP）の最適化（推奨）

URLを送信すると、多くのメッセージアプリで自動的にプレビュー（画像、タイトル、説明文）が表示されます。これを最適化することで、視覚的に魅力的なメッセージにできます。

**実装方法：**
- Open Graph（OG）メタタグを追加
- イベントごとに動的なOG画像を生成
- 魅力的な説明文を設定

### 2. 短縮URL + QRコード

- 短縮URLを生成して、タップしやすくする
- QRコードを画像として送信（LINEやFacebookで読み取れる）

### 3. メッセージ文面の工夫

URLと一緒に送るメッセージを工夫して、タップしたくなるようにする：

```
【イベント告知】🎉

[イベント名]
日時: [日時]
場所: [場所]

参加したい気持ちを教えてください！
👇タップしてスライダーで入力👇
[URL]
```

## 実装可能な改善

### OGPメタタグの追加

`/embed/[slug]`ページに動的なOGPメタタグを追加することで、URLプレビューを最適化できます。

**メリット：**
- URLを送信すると、イベント情報が自動的に表示される
- 画像、タイトル、説明文が表示される
- タップしやすくなる

**制限：**
- スライダー自体はメッセージ内に表示されない
- リンクをタップする必要がある

## 結論

**現時点では、以下の方法が最も現実的です：**

1. **URLを送信** → メッセージアプリでプレビューが表示される
2. **受信者がリンクをタップ** → スライダーページが開く
3. **スライダーを操作** → 気持ちを送信

OGPメタタグを最適化することで、URLプレビューが魅力的になり、タップ率が向上します。

将来的に、Discord BotやSlack Botを開発する場合は、インタラクティブなコンポーネントを送信できますが、Botアカウントが必要です。
