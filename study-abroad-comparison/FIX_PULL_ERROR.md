# 「チェックアウトの前にリポジトリの作業ツリーを消去してください」エラーの解決方法

## 🔍 エラーの原因

このエラーは、ローカルに未コミットの変更がある場合に発生します。

---

## ✅ 解決方法

### 方法1: 変更をコミットしてからPull（推奨）

#### ステップ1: 未コミットの変更を確認

1. **ソース管理パネル（📊）を開く**
   - `Ctrl + Shift + G` を押す

2. **「Changes」セクションを確認**
   - 変更されたファイルが表示されます

#### ステップ2: 変更をステージングしてコミット

1. **すべての変更をステージング**
   - 「Stage All Changes」をクリック
   - または `Ctrl + Shift + P` → 「Git: Stage All Changes」

2. **コミット**
   - コミットメッセージを入力して `Ctrl + Enter`
   - または `Ctrl + Shift + P` → 「Git: Commit」

#### ステップ3: Pullを実行

1. `Ctrl + Shift + P` → 「**Git: Pull**」
2. Pullが完了したら、Pushを実行

---

### 方法2: 変更を一時的に保存（Stash）してからPull

変更をコミットしたくない場合：

1. **コマンドパレットを開く**（`Ctrl + Shift + P`）

2. **変更を一時保存**
   - 「**Git: Stash**」と入力して選択
   - または「**Git: Stash (Include Untracked)**」を選択

3. **Pullを実行**
   - `Ctrl + Shift + P` → 「**Git: Pull**」

4. **保存した変更を復元**（必要に応じて）
   - `Ctrl + Shift + P` → 「**Git: Stash Pop**」

---

### 方法3: リモートリポジトリが空の場合、強制的にPush

もしGitHubリポジトリが空で、ローカルの変更を優先したい場合：

1. **まず、ローカルの変更をすべてコミット**
   - ソース管理パネルで変更をステージングしてコミット

2. **強制的にPush**
   - `Ctrl + Shift + P` → 「**Git: Push (Force)**」
   - ⚠️ 注意: これはリモートの内容を上書きします

---

## 🎯 推奨手順

**最も安全な方法：**

1. ✅ すべての変更をステージング
2. ✅ コミット
3. ✅ Pullを実行（マージコンフリクトがある場合は解決）
4. ✅ Pushを実行

---

## 📝 確認事項

ソース管理パネル（📊）で確認：
- 「Changes」セクションにファイルがありますか？
- 「Staged Changes」セクションにファイルがありますか？

ファイルがある場合は、まずコミットしてください。
